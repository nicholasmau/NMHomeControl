# Contributing to NM Home Control

Thank you for your interest in contributing to NM Home Control! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/NMHomeControl.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes thoroughly
6. Commit your changes: `git commit -m "Description of your changes"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Create a Pull Request

## Development Setup

See [QUICKSTART.md](QUICKSTART.md) for detailed setup instructions.

## Code Style

- **Backend (TypeScript)**: Follow existing code patterns, use ESLint
- **Frontend (React/TypeScript)**: Follow React best practices, use Tailwind CSS for styling
- **Commits**: Write clear, descriptive commit messages

## Testing

- Run backend tests: `cd backend && npm test`
- Ensure all tests pass before submitting PR
- Add tests for new features

## Security

- Never commit sensitive data (API tokens, passwords, etc.)
- Follow security best practices outlined in [SECURITY_AUDIT.md](SECURITY_AUDIT.md)
- Report security vulnerabilities privately (see Security Policy below)

## Pull Request Guidelines

- Keep PRs focused on a single feature or fix
- Update documentation as needed
- Ensure all tests pass
- Follow the existing code style
- Provide a clear description of your changes

## Questions?

Open an issue for questions or discussions about the project.
